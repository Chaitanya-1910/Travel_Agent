AI Travel Planner — Complete UI Design Brief

Built with Flask + IBM Watsonx.ai (Mistral Small 24B) · Full-stack web app · 4 pages · Dark/Light mode

Project Overview

A full-stack AI-powered travel planning web application. Users enter their trip preferences and 10 specialised AI agents (powered by IBM Watsonx.ai) simultaneously generate a complete travel plan — itinerary, budget, hotels, transport, weather, local guide, restaurants, and packing checklist. The app also features a real-time AI chat assistant.

Flask Backend IBM Watsonx.ai 10 AI Agents Bootstrap 5 PDF Export Dark / Light Mode Responsive

Pages (4 Total)

1. Dashboard / Home
/ → index.html

The landing page and main dashboard. Sections from top to bottom:

  • Hero Section — Full-width gradient background with animated floating shapes. Left: badge ("Powered by IBM Watsonx.ai"), large H1 "Your AI Travel Companion", subtitle, two CTA buttons (Plan My Trip / Chat with AI), and 3 quick stats (10+ Agents, 190+ Destinations, ∞ Possibilities). Right: floating animated preview card showing a sample Tokyo trip with day-wise activities.
  • AI Agents Section — 10-card grid. Each card has a coloured icon, agent name, description, and "Get Started" link. Cards use a CSS variable --agent-color for per-card accent colouring. Reveal-on-scroll animation.
  • Dashboard Widgets — Two-column layout: Left = "Recent & Upcoming Trips" widget (loads real trip data from API, shows country flag emoji, destination, dates, status badge — Completed/Ongoing/Upcoming/Planned, budget chip, travellers chip, AI insight callout with blue left-border). Right = "Quick Plan" mini-form (destination input, duration select, budget input, Plan with AI button).
  • Stats Row — 4 stat cards: Trips Planned (animated counter), Favourites (animated counter), AI Agents (10), Destinations (190+). Each has a gradient icon circle.
  • Popular Destinations — 8-card grid with gradient backgrounds, large flag emoji, destination name, estimated cost, and "Plan This Trip" button that pre-fills the planner.
  • How It Works — 3 steps: Tell Us Your Dream → AI Does the Magic → Travel & Enjoy. Step number in large muted text behind icon.
  • CTA Banner — Dark gradient section with "Ready for Your Next Adventure?" and two buttons.
2. Trip Planner
/planner → planner.html

Two-column layout (form left, results right).

  • Left — Planner Form: Destination input, From Location input, Start/End date pickers (end auto-fills from duration), Duration select, Travellers number input, Budget input with preset buttons (Budget / Moderate / Luxury), Interest tags (toggleable pill buttons: Culture, Food, Nature, Adventure, etc.), Travel Style select, Accommodation select, Dietary select. Submit button "Plan My Trip with AI".
  • Right — Three States:
    • Empty State: Illustrated icon + "Your AI-Powered Trip Plan Will Appear Here" message.
    • Loading State: Animated paper-plane with trail, "Your AI Travel Team is Working" title, live elapsed timer (counts up in seconds/minutes), "Usually completes in 60–120s" hint, and 8 agent progress rows each showing clock → spinner → green checkmark as agents complete sequentially.
    • Results State: Trip header with destination name, meta chips (duration, travellers, budget, style). Tab navigation (Itinerary / Budget / Hotels / Transport / Weather / Local Guide / Restaurants / Packing). Tab content rendered as formatted markdown with headings, bullets, tables. Action buttons: Save Trip (bookmark toggle) and Export PDF.
3. AI Chat Assistant
/chat → chat.html
  • Left sidebar — "AI Travel Assistant" header with robot icon and online indicator. Suggested question chips. Conversation history list.
  • Right chat panel — Chat messages in bubble style. User messages: right-aligned purple gradient bubbles with avatar. AI messages: left-aligned white/dark card with robot avatar and timestamp. Typing indicator (animated dots) while waiting for response.
  • Input area — Full-width textarea with character counter (0/2000), send button with spinner state. Supports Enter to send.
  • Welcome message on load with bullet list of capabilities.
4. Saved Trips
/trips → trips.html
  • Header with trip count badge and filter buttons (All / Saved / Favourites).
  • Trip cards grid — each card shows destination, dates, duration, travellers, budget, status badge, favourite heart toggle, and action buttons (View Details, Export PDF, Delete).
  • Trip detail modal — opens on click, shows full AI results with same tab navigation as planner results.
  • Empty state with illustration when no trips exist.

Shared Components (base.html)

  • Navbar — Fixed top, transparent → frosted glass on scroll. Logo left, nav links centre (Dashboard, Planner, Chat, My Trips), theme toggle button (sun/moon icon) right. Hamburger on mobile.
  • Toast Notifications — Bootstrap toast, bottom-right. Three types: success (green), error (red), info (blue).
  • Footer — Dark background, logo, tagline, nav links, tech stack badges (IBM Watsonx.ai, IBM Granite, Flask, Bootstrap 5).
  • Scroll reveal — IntersectionObserver-based fade-in-up on agent cards, destination cards, stat cards, how-it-works cards.

Tech Stack

Backend

  • Python 3 + Flask 3.1
  • IBM Watsonx.ai SDK (ibm-watsonx-ai)
  • Model: mistralai/mistral-small-3-1-24b-instruct-2503
  • 10 specialised agent functions, sequential execution with rate-limit retry
  • PDF export via fpdf2
  • In-memory trip store (dict)
  • REST API: /api/plan-trip, /api/chat, /api/trips, /api/health

Frontend

  • Jinja2 templates (base.html + 4 pages)
  • Bootstrap 5 (layout/utilities only)
  • Custom CSS (~1200 lines, CSS variables for theming)
  • Vanilla JS (no framework): main.js, dashboard.js, planner.js, chat.js, trips.js
  • Font Awesome 6 icons
  • Google Fonts (Inter)
  • Dark/light mode via data-theme attribute + localStorage

File Structure

  • app.py — Flask app, all REST endpoints
  • agents/travel_agents.py — All 10 AI agent functions + WatsonxClient singleton + TravelAgentOrchestrator
  • utils/ — PDF generator, input sanitiser, trip ID generator
  • templates/ — base.html, index.html, planner.html, chat.html, trips.html
  • static/css/style.css — All custom styles (~1200 lines)
  • static/js/ — main.js, dashboard.js, planner.js, chat.js, trips.js
  • .env — IBM_API_KEY, IBM_PROJECT_ID, IBM_URL, IBM_MODEL_ID, FLASK_SECRET_KEY
