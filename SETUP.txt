==============================================================================
  TravelAI — Setup Guide
  AI-Powered Travel Planner using IBM Watsonx.ai
==============================================================================

PREREQUISITES
-------------
  - Python 3.9 or higher  (python --version to check)
  - pip  (comes with Python)
  - An IBM Cloud account with Watsonx.ai access (free tier works)
  - Git  (to clone the repository)


------------------------------------------------------------------------------
STEP 1 — Clone the Repository
------------------------------------------------------------------------------

  git clone https://github.com/<your-username>/Travel_Agent.git
  cd Travel_Agent


------------------------------------------------------------------------------
STEP 2 — Create a Virtual Environment
------------------------------------------------------------------------------

  Windows:
    python -m venv venv
    venv\Scripts\activate

  Mac / Linux:
    python -m venv venv
    source venv/bin/activate

  You should see (venv) in your terminal prompt after activation.


------------------------------------------------------------------------------
STEP 3 — Install Dependencies
------------------------------------------------------------------------------

  pip install -r requirements.txt


------------------------------------------------------------------------------
STEP 4 — Get Your IBM Watsonx.ai Credentials  (REQUIRED)
------------------------------------------------------------------------------

  You need THREE values from IBM Cloud:

  A) IBM_API_KEY
     1. Go to https://cloud.ibm.com
     2. Click your profile (top right) -> Manage -> Access (IAM)
     3. Click "API Keys" in the left menu -> "Create"
     4. Give it a name, click Create, then COPY the key immediately
        (you cannot see it again after closing the dialog)

  B) IBM_PROJECT_ID
     1. Go to https://dataplatform.cloud.ibm.com
     2. Open your Watsonx.ai project (or create one)
     3. Click "Manage" tab -> "General"
     4. Copy the "Project ID" shown on that page

  C) IBM_URL  — MUST match the region of your Watson Machine Learning SERVICE:
     - US South (Dallas)    : https://us-south.ml.cloud.ibm.com
     - EU DE    (Frankfurt) : https://eu-de.ml.cloud.ibm.com
     - EU GB    (London)    : https://eu-gb.ml.cloud.ibm.com
     - AP North (Tokyo)     : https://jp-tok.ml.cloud.ibm.com

     To find your WML service region:
       cloud.ibm.com -> Resource List -> AI / Machine Learning
       -> Watson Machine Learning -> look at the Region/Location column

  IMPORTANT: Your Watsonx Studio (project) region and your WML service region
             CAN be different. IBM_URL must match the WML SERVICE, not the project.
             Wrong region = all API calls fail with auth/routing errors.


------------------------------------------------------------------------------
STEP 5 — Create Your .env File
------------------------------------------------------------------------------

  Windows:
    copy .env.example .env

  Mac / Linux:
    cp .env.example .env

  Now open the .env file in any text editor and fill in the three values:

    IBM_API_KEY=<paste your API key here>
    IBM_PROJECT_ID=<paste your project ID here>
    IBM_URL=<paste your regional URL here>

  Everything else in the file already has sensible defaults.

  IMPORTANT: Never share or commit your .env file — it contains your secrets.
             The .gitignore already excludes it from Git automatically.


------------------------------------------------------------------------------
STEP 6 — Run the Application
------------------------------------------------------------------------------

  python app.py

  Then open your browser and go to:

    http://localhost:5000


------------------------------------------------------------------------------
TROUBLESHOOTING
------------------------------------------------------------------------------

  "token_quota_reached" (403 error)
    -> Your IBM free-tier monthly token quota is exhausted.
       Option 1: Wait for it to reset on the 1st of next month.
       Option 2: Upgrade to Pay-As-You-Go at cloud.ibm.com -> Billing.
       Option 3: Change IBM_MODEL_ID in .env to a different model and retry.

  "Model is not supported" or model errors
    -> Your IBM_URL region may not match where your Watsonx project lives.
       Also try changing IBM_MODEL_ID in .env to one of these alternatives:
         meta-llama/llama-3-3-70b-instruct          (recommended)
         meta-llama/llama-4-maverick-17b-128e-instruct-fp8
         mistralai/mistral-small-3-1-24b-instruct-2503

  "IBM_API_KEY and IBM_PROJECT_ID must be set"
    -> The .env file was not found or the values are still blank.
       Re-check Step 5 and make sure the .env file is in the project root.

  "ModuleNotFoundError"
    -> Your virtual environment may not be activated.
       Run:  venv\Scripts\activate  (Windows)
         or  source venv/bin/activate  (Mac/Linux)
       Then re-run:  pip install -r requirements.txt

  Port already in use
    -> Change FLASK_PORT in your .env file to another port, e.g. 5001,
       then visit http://localhost:5001


------------------------------------------------------------------------------
SUPPORTED AI MODELS (IBM Watsonx.ai)
------------------------------------------------------------------------------

  The app is pre-configured to use:
    meta-llama/llama-3-3-70b-instruct   ← recommended, best quality

  Other models you can set via IBM_MODEL_ID in .env:
    - meta-llama/llama-4-maverick-17b-128e-instruct-fp8
    - mistralai/mistral-small-3-1-24b-instruct-2503
    - meta-llama/llama-3-1-70b-gptq

  Each IBM account has its own free-tier token quota per model.
  If one model hits the quota limit, try switching to another.
